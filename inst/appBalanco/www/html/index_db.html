Padronizações adotadas

Modelo de Banco de Dados adotado

• Para modelar os dados de entrada da ferramenta foi utilizado o padrão de Banco de Dados Relacionais.

• Um banco de dados relacional é um banco que modela os dados de uma forma que eles sejam percebidos pelo usuário como tabelas, ou mais formalmente relações


Tabelas Dimensão e Fato

• Utilizamos o padrão de tabelas de dimensão e fato no nosso banco

• As tabelas de dimensão são tabelas que identificam caracteristicas unicas em cada linha, é muito importante para evitar a repetição de informações na base de dados

• As tabelas de fato são tabelas que armazenam falores detalhados de medidas ou fatos em cada linha


Nomenclaturas

• Tabelas de dimensão começam com dim seguido do nome a que se referem. Exemplo: dim_torre_ama

• Tabelas de fato começam com fat seguido do nome a que se referem. Exemplo: fat_comparacao_torre_ama

• Para nomes de colunas usa-se cd_ para campos de código, vr_ para campos de valor, dt_ para datas em qualquer formato, nm_ para nomes,frases,links e etc. e tip_ para tipo (indicadores, como veradeiro ou falso, 0 ou 1)


Tabela da base SQLite

fat_comparacao_torre_ama

A tabela tem como função principal indicar as duas estações de comparação para verificação de medições suspeitas, assim como guardar todos os valores de τ e de tolerância para as regressões quantílicas que definirão o intervalo de confiança para valores válidos.

Campos:

‣ cd_ama: código da estação AMA, chave primaria da tabela (PK)

‣ vr_anemometro_intermediario: identificador de anemômetro intermediário, 1 se possui, 0 se não

‣ cd_ama_comp_1: código da estação AMA de comparação 1

‣ cd_ama_comp_2: código da estação AMA de comparação 2

‣ dt_inicio_comparacao_lp: data de início do período de histórico de referência no formato AAAAMMDD

‣ dt_fim_comparacao_lp: data de fim do período de histórico de referência no formato AAAAMMDD

‣ dt_ultima_analise: data do último dado verificado no formato AAAAMMDD

‣ tip_processado: identificador de processamento formato 0/1. Usado somente para processamento paralelo

‣ vr_tau_sup_*_**: quantil superior para comparação entre os instrumentos * e ** (ex. vr_tau_sup_asupmed_aintmed)

‣ vr_tau_inf_*_**: quantil inferior para comparação entre os instrumentos * e ** (ex. vr_tau_sup_asupmed_comp_2)

‣ vr_tol_sup_quantil_*_**: valor de tolerância da curva do quantil superior para comparação entre os instrumentos * e **

‣ vr_tol_inf_quantil_*_**: valor de tolerância da curva do quantil inferior para comparação entre os instrumentos * e **
