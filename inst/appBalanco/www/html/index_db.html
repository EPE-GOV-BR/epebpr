Estrutura da Base de Dados SQLite


Tabelas da base SQLite

BPO_A01_CASOS_ANALISE

A tabela possui as informações dos casos executados no balanço de ponta.

Campos:

‣ A01_TP_CASO  : Tipo de caso (1:PDE)

‣ A01_NR_CASO  : Número do caso. Identificador numérico do caso definido pelo usuário

‣ A01_CD_MODELO  : Tipo de modelo dos dados de entrada (1= Newave, 2=Suishi)

‣ A01_TX_DESCRICAO: Descrição do caso

‣ A01_NR_MES_INICIO: Mês Início do BP, no formato aaaamm

‣ A01_NR_MES_FIM: Mês Final do BP, no formato aaaamm

‣ A01_NR_HORAS_PONTA: Número de horas considerada para modulação da geração hidráulica para o atendimento de ponta

‣ A01_NR_COTA_LIMITE_TUCURUI: Cota Tucuruí a qual ocorre limitação da Geração (Simulando a perda da segunda casa de força [m]

‣ A01_NR_GERACAO_LIMITE_TUCURUI: Geração Limite de Tucuruí para a cota limite informada [MWmed]

‣ A01_VL_RESERVA_OPERATIVA: Valor de reserva operativa que será acrescida à demanda

‣ A01_IN_DEMANDA_LIQUIDA: Tipo de Demanda: 1 para Demanda Líquida, 0 para Determinística

‣ A01_NR_SERIES_HIDRO: Total de séries hidrológicas

‣ A01_NR_MES_INICIO_MDI: Ano mês inicial de expansão no arquivo Dados_MDI_PDE_AAAA, no formato aaaamm

‣ A01_NR_MES_FIM_MDI: Ano mês final de expansão no arquivo Dados_MDI_PDE_AAAA, no formato aaaamm


BPO_A02_REES

A tabela possui as informações dos reservatórios equivalentes de energia utilizados.

Campos:

‣ A01_TP_CASO  : Tipo de caso (1:PDE)

‣ A01_NR_CASO  : Número do caso. Identificador numérico do caso definido pelo usuário

‣ A01_CD_MODELO  : Tipo de modelo dos dados de entrada (1= Newave, 2=Suishi)

‣ A02_NR_REE  : Código do REE

‣ A02_TX_DESCRICAO_REE: Nome do REE

‣ A02_TP_CALC_POTENCIA: Tipo de Modução Geração Hidro: 1-modula a geração hidro na média (Saída Modelo de Simulação) para atender às x horas de ponta; 2-não modula e considera a geração hidráulica da ponta resultante da simulação; 3-não modula e considera a geração hidráulica média resultante da simulação

‣ A02_NR_SUBSISTEMA: número do subsistema/submercado
