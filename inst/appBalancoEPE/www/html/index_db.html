Tabelas da base SQLite

BPO_A01_CASOS_ANALISE

A tabela possui as informações dos casos executados no balanço de ponta.

Campos:

‣ A01_TP_CASO  : Tipo de caso (1:PDE)

‣ A01_NR_CASO  : Número do caso. Identificador numérico do caso definido pelo usuário

‣ A01_CD_MODELO  : Tipo de modelo dos dados de entrada (1= Newave)

‣ A01_TX_DESCRICAO: Descrição do caso

‣ A01_NR_MES_INICIO: Mês Início do BP, no formato aaaamm

‣ A01_NR_MES_FIM: Mês Final do BP, no formato aaaamm

‣ A01_NR_HORAS_PONTA: Número de horas considerada para modulação da geração hidráulica para o atendimento de ponta

‣ A01_NR_COTA_LIMITE_TUCURUI: Cota Tucuruí a qual ocorre limitação da Geração (Simulando a perda da segunda casa de força [m]

‣ A01_NR_GERACAO_LIMITE_TUCURUI: Geração Limite de Tucuruí para a cota limite informada [MWmed]

‣ A01_VL_RESERVA_OPERATIVA: Valor de reserva operativa que será acrescida à demanda

‣ A01_IN_DEMANDA_LIQUIDA: Tipo de Demanda: 0 para Demanda Determinística

‣ A01_NR_SERIES_HIDRO: Total de séries hidrológicas

‣ A01_NR_MES_INICIO_MDI: Ano mês inicial de expansão no arquivo Dados_MDI_PDE_AAAA, no formato aaaamm

‣ A01_NR_MES_FIM_MDI: Ano mês final de expansão no arquivo Dados_MDI_PDE_AAAA, no formato aaaamm

BPO_A08_DADOS_CALCULADOS_UHE

A tabela possui os dados das usinas hidroelétricas calculados pela metodologia de rateio da disponibilidade hídrica.

Campos:

‣ A01_TP_CASO  : Tipo de caso (1:PDE)

‣ A01_NR_CASO  : Número do caso. Identificador numérico do caso definido pelo usuário

‣ A01_CD_MODELO  : Tipo de modelo dos dados de entrada (1= Newave)

‣ A03_CD_USINA  : Código da usina no cadastro de usinas hidroelétricas

‣ A08_NR_MES  : Ano mês no formato aaaamm

‣ A08_NR_SERIE  : Número da série hidro

‣ A02_NR_REE: Código do REE

‣ A08_VL_VOLUME_OPERATIVO: Volume operativo calculado a partir dos volumes máximo, e mínimo de cada usina e do percentual do armazenamento do reservatório equivalente de energia no qual a usina se encontra [hm3]

‣ A08_VL_COTA_OPERATIVA: Cota operativa calculada ao aplicar o volume operativo no polinômio cota-volume [m]

‣ A08_VL_ALTURA_LIQUIDA: Altura de queda líquida calculada a partir da cota operativa, do canal de fuga médio e perdas [m]

‣ A08_VL_VAZAO_MAXIMA: Valor de vazão máxima - Não aplicável ao modelo NEWAVE

‣ A08_VL_POTENCIA_MAXIMA: : Potência máxima disponível, após perda de potência por deplecionamento dos reservatórios, calculada a partir da altura de queda líquida, da altura de referência de queda e do tipo de turbina da usina [MW]

‣ A08_VL_GERACAO_HIDRO_MINIMA_ORIGINAL: Geração hidráulica mínima de cada usina, calculada a partir de sua restrição de vazão mínima [MWmes]

‣ A08_VL_GERACAO_HIDRO_MINIMA: Geração hidráulica mínima de cada usina, ajustada a partir da conferência com os dados totais de geração hidráulica do NEWAVE, para o REE para qual a usina estáu inserida [MWmes]

‣ A08_VL_GERACAO_HIDRO_MEDIA: Geração hidráulica média mensal de cada usina, calculada a partir de rateio dos dados totais de geração hidráulica do NEWAVE, para o REE para qual a usina está inserida, de acordo com a potência instalada de cada usina e seu respectivo valor de geração hidráulica mínima [MWmes]

‣ A08_VL_VAZAO_MAXIMA_MODULADA: Valor de vazão máxima modulada - Não aplicável ao modelo NEWAVE

‣ A08_VL_ALTURA_MODULADA: Valor de altura modulada - Não aplicável ao modelo NEWAVE

‣ A08_VL_POTENCIA_MAXIMA_MODULADA: Potência disponível máxima, calculada após modulação da geração hidráulica média, mantendo a restrição de vazão mínima fora da ponta, limitada para potência disponível máxima calculada para a perda de potência por deplecionamento [MW]
